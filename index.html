Introduccion al Software Libre y GNU/Linux
Administracion de Sistemas + VirtualBox + Proyecto
▶ Teoria TEORIA

Generaciones de los Sistemas Operativos

  • 1a Generacion (1945-1955): Ordenadores voluminosos, tarjetas perforadas. No existia SO, el operario introducia el codigo.
  • 2a Generacion (1955-1968): Transistores, JCL, lenguajes de alto/bajo nivel, superusuarios, dispositivos E/S.
  • 3a Generacion (1968-1981): Circuitos integrados (LSI), equipos de proposito general, multiprogramacion, buffering, tiempo compartido.
  • 4a Generacion (1981-2001): VLSI, microprocesadores, ordenador personal, Plug and Play, cliente/servidor, maquinas virtuales.
  • 5a Generacion (2001-hoy): Tendencia a que personas no expertas manejen ordenadores.

Sistemas Operativos Historicos y Actuales

  • UNIX: Multitarea, multiusuario. Laboratorios Bell de AT&T.
  • Mac OS: Primer SO de Apple para Macintosh.
  • Windows: SO grafico para PCs de Microsoft.
  • Cuota actual: Android ~46%, Windows ~25%, iOS ~18%, OS X ~5%, Linux ~1.3%.

Software Libre

Historia

  • Anos 60-70: Pocas computadoras (mainframes), software artesanal, codigo fuente compartido libremente.
  • Anos 80: Software privativo se hace fuerte. Richard Stallman en 1984 comienza el proyecto GNU. En 1985 funda la Free Software Foundation (FSF).
  • Anos 90: En 1991 Linus Torvalds crea el primer nucleo de GNU/Linux.

Las 4 Libertades del Software Libre

  • Libertad 0: Usar el programa, con cualquier proposito.
  • Libertad 1: Estudiar como funciona y adaptarlo.
  • Libertad 2: Distribuir copias.
  • Libertad 3: Mejorar el programa y hacer publicas las mejoras.

Copyleft y Licencias

Copyleft: restriccion que impide distribuir copias o modificaciones restringiendo las 4 libertades.

Con CopyleftSin Copyleft
GPL/AGPL, MPL, CC-saBSD, MIT, CC

Que NO es software libre: Software de dominio publico, semilibre, freeware (Edge, Adobe Reader), shareware (WinRAR), software con fuentes.

GNU/Linux

  • 1991: Linus Torvalds presenta la version 0.01 del kernel.
  • 1992: Se libera bajo licencia GPL.
  • 1994: Version 1.0. 2003: Version 2.6 (~6M lineas). 2008: Android basado en Linux.
  • Es un SO libre (reimplementacion de UNIX, estandar POSIX), gratuito, mantenido por la comunidad.

El Sistema Informatico y la Organizacion

El Departamento de Informatica se encarga de mantener y gestionar el Sistema Informatico:

  • Hardware: Servidores, PCs, cableado, centro de datos.
  • Software: SO, aplicaciones empresariales, aplicaciones especificas.
  • Personal: Responsable de Informatica + Personal Tecnico.

Funciones: Admin servidores, admin usuarios, admin red, admin datos, admin web, seguridad, desarrollo.

Rol del Administrador de Sistemas

Persona responsable de implementar, configurar, mantener, monitorizar, documentar y asegurar el sistema informatico.

Cualidades: Autoridad + responsabilidad + servicio + cooperacion.

Tareas Detalladas

  • Hardware: Entorno fisico, cortes de suministro, reparar componentes, redes.
  • Software: Instalar SO, detectar problemas, configurar aplicaciones, documentar.
  • Soporte: Formar usuarios, proporcionar soporte, sistema de rastreo (tickets).
  • Servicios: Cuentas de usuario, politicas, servicio de nombres, servidores web.
  • Seguridad: Cuotas de disco, logs, firewalls, alertas CERT.
  • Backups: Estrategia, software, automatizar, planes de supervivencia.

Estrategia General

  1. Planear antes de hacer los cambios.
  2. Hacer los cambios reversibles.
  3. Realizar cambios de forma incremental.
  4. Probar, probar, probar.
  5. Conocer realmente como trabajan las cosas.

Organizacion de un SO tipo GNU/Linux
Ficheros, permisos, enlaces, estructura FHS
▶ Teoria TEORIA

Ficheros en GNU/Linux

"Si algo no es un fichero, entonces es un proceso"
  • Programas: /bin/ls, /usr/bin/find
  • Dispositivos I/O: /dev/sda, /dev/tty0
  • Comunicacion entre procesos: sockets, pipes
  • Estructura jerarquica: / = directorio raiz

Nodos-i

Nodos-i = metadatos de los ficheros: tamano, permisos, posicion sectores, numero de enlaces. Cada fichero tiene un nodo-i unico. Ver con ls -i o df -i.

Gestion del Acceso: Propietarios y Permisos

Cada fichero tiene dos propietarios: usuario y grupo.

chown javi fichero # Cambiar usuario propietario chown javi:profesores fichero # Cambiar usuario:grupo chgrp profesores fichero # Cambiar grupo

Permisos rwx

AccesoFicheroDirectorio
rVer contenidoListar contenido
wModificar contenidoCrear/eliminar ficheros
xEjecutar ficheroEntrar (cd) en el directorio
chmod u+r fichero # Modo simbolico chmod u=rwx,go= fichero # Modo simbolico chmod -R u+rwx,go-rwx dir # Recursivo chmod 740 fichero # Modo absoluto: u+rwx,g+r,o-rwx

Permisos Especiales

  • suid (chmod u+s): en ejecutables, se ejecuta como el propietario del fichero. Ej: /usr/bin/passwd tiene -rwsr-xr-x.
  • sgid (chmod g+s): en ejecutables, grupo efectivo = grupo del fichero. En directorios, nuevos ficheros heredan el grupo del directorio.
  • sticky bit (chmod o+t): en directorios, solo root o el propietario pueden borrar/renombrar. Ej: /tmp.

Mascara de Permisos (umask)

Base: directorios 777, ficheros 666. La mascara resta permisos.

umask # Ver mascara actual (ej: 0022) umask 027 # Cambiar: dir=750, fich=640

Tipos de Ficheros

  • Normal (-), Directorio (d), Bloque (b), Caracter (c)
  • Named Pipes (p), Socket (s), Enlace simbolico (l)

Enlaces

Enlace Fisico (hard link)

  • Dos nombres apuntando al mismo nodo-i. ln archivo enlace
  • Solo dentro de la misma particion. No se puede hacer a directorios.

Enlace Simbolico (symlink)

  • Puntero virtual (acceso directo). ln -s archivo enlace
  • Nodo-i independiente, contiene la ruta del archivo.

Dispositivos

  • /dev/sda = primer disco duro, /dev/sda1 = primera particion
  • /dev/tty1 = primera terminal, /dev/nvme0n1 = disco NVMe

Estructura FHS (Filesystem Hierarchy Standard)

DirectorioContenido
/binEjecutables basicos (mv, cp)
/devFicheros especiales de dispositivos
/etcConfiguracion local del sistema
/homeDirectorios de trabajo de usuarios
/libBibliotecas necesarias
/proc, /sysSF virtuales (procesos, nucleo)
/tmpFicheros temporales (sticky bit)
/varDatos variables (logs, spool, mail)
/bootNucleo y configuracion arranque
/usrSubdirs de solo lectura (bin, sbin, lib, share, local, src)
/rootHOME del administrador
/optAplicaciones estaticas
/mnt, /mediaMontaje de otros SF

Estaticos: /bin, /sbin, /opt, /boot. Dinamicos: /var, /home. Compartibles: /usr/bin, /opt. No compartibles: /etc, /boot, /var/run.

03
Gestion de Usuarios
passwd, shadow, grupos, contrasenas, herramientas
▶TeoriaTEORIA

Ficheros Basicos

  • /etc/passwd - Cuentas de usuarios (rw-r--r--).
  • /etc/shadow - Hashes de contrasenas (rw-r----- root:shadow).
  • /etc/group - Grupos y miembros.
  • /etc/gshadow - Contrasenas de grupos cifradas.

Formato /etc/passwd

nombre:password:uid:gid:gecos:home:shell tux:x:1001:100:The Linux penguin:/home/tux:/bin/bash

password: "!!" = bloqueada, "*" = desactivada, "x" = shadow activas.

Shadow Passwords

Contrasenas cifradas en /etc/shadow con hash + salt. Formato: $id[$params]$salt$hash

IDAlgoritmo
$1$MD5 (128 bits, obsoleto)
$5$SHA-256
$6$SHA-512
$y$yescrypt (moderno, recomendado)

Restricciones de Tiempo (/etc/shadow)

Formato: nom:pass:changed:minlife:maxlife:warn:inactive:expired:unused

chage -M 20 pagutierrez # Max dias sin cambiar pass chage -W 6 pagutierrez # Aviso dias antes de expirar chage -I 5 pagutierrez # Dias inactividad tras expirar chage -E 2026-10-30 pagutierrez # Fecha expiracion cuenta chage -l usuario # Consultar parametros

Ficheros de Inicializacion

/etc/skel/ = ficheros copiados automaticamente a cada $HOME nuevo.

Operacionbash
login.bash_profile + .profile
ejecuta shell.bashrc
logout.bash_logout

Cuentas Restrictivas

  • Shell = fichero ejecutable: al entrar se ejecuta, al terminar sale.
  • rbash (/bin/rbash): no puede cambiar directorio, ni modificar $PATH, ni usar redirecciones.
  • Asignar /bin/false o /sbin/nologin para impedir login.

Anadir Usuarios al Sistema

  1. Decidir nombre, UID, grupos (primario + secundarios).
  2. Introducir datos en /etc/passwd y /etc/group.
  3. Asignar password.
  4. Escribir contrasena en shadow (pwck).
  5. Establecer parametros de envejecimiento.
  6. Crear $HOME, propietario y permisos.
  7. Copiar ficheros desde /etc/skel/.
adduser usuario # Crear usuario (interactivo) useradd usuario # Crear (mas bajo nivel) usermod -aG grupo user # Anadir a grupo deluser usuario # Eliminar (no borra $HOME)

Grupos

Colecciones de usuarios que comparten recursos. Fichero: /etc/group (nombre:x:gid:lista usuarios).

  • Primario: especificado en /etc/passwd. Secundarios: en /etc/group.
  • Grupo activo: al crear ficheros, se usa el grupo activo (normalmente el primario).
addgroup grupo groupmod grupo delgroup grupo groups [usuario] id [usuario] newgrp grupo

Usuarios Estandar

  • uid 0-99: usuarios del propio SO (root=0).
  • uid 100-499: servicios o programas (bin, daemon, mail, ftp, postgres).
  • uid >= 1000: usuarios normales.
  • nobody/nfsnobody: usuario sin privilegios (NFS).

  3. Cambia la caducidad y prueba el resultado.

04
Arranque y Parada del Sistema y Servicios
Boot, GRUB, init, systemd, niveles de ejecucion
▶TeoriaTEORIA

Proceso de Arranque

Power ON → BIOS/UEFI → Boot Loader (GRUB) → Kernel → initrd → init/systemd (PID=1) → Sistema operativo

BIOS vs UEFI

  • BIOS: BIOS → MBR → Boot Loader → Kernel → OS
  • UEFI: UEFI → EFI Boot Loader → Kernel → OS

Programa Cargador (MBR)

En los primeros sectores del disco. Carga el nucleo del SO.

initrd (init RAM Disk)

Contiene modulos del nucleo necesarios para leer del disco. El nucleo carga primero el initrd, luego los modulos, y continua el arranque.

GRUB (GRand Unified Bootloader)

  • Configuracion: /etc/default/grub
  • Scripts: /etc/grub.d/
  • Archivo generado: /boot/grub/grub.cfg (NO editar a mano)
  • Regenerar: sudo update-grub2
GRUB_DEFAULT=0 GRUB_TIMEOUT=10 GRUB_CMDLINE_LINUX_DEFAULT="quiet splash" GRUB_TERMINAL=console GRUB_BACKGROUND="/boot/bg.png"

Modo Monousuario vs Multiusuario

  • Monousuario: tareas administrativas, solo monta /, pocos demonios, shell como root.
  • Problema! Si en GRUB ponemos init=/bin/bash, no se llama a sulogin. Solucion: contrasena en GRUB.

Niveles de Ejecucion

Nivelsystemd targetDescripcion
0poweroff.targetSistema apagado
1, srescue.targetMonousuario
2-4multi-user.targetMultiusuario con red
5graphical.targetMultiusuario + GUI
6reboot.targetReiniciandose

systemd

systemctl start|stop|restart|status servicio systemctl enable|disable servicio systemctl list-units --type=service systemctl set-default multi-user.target systemctl isolate rescue.target journalctl -xe # Ver logs

Parada del Sistema

shutdown -h now # Apagar ahora shutdown -r +5 # Reiniciar en 5 min shutdown -c # Cancelar

Procesos en GNU/Linux

Proceso: programa en ejecucion. La CPU ejecuta un proceso a la vez por nucleo, conmutando rapidamente (cuanto ~100ms).

Modos de Ejecucion

  • Modo usuario: codigo normal del programa.
  • Modo nucleo: llamadas al sistema, excepciones, interrupciones.

Tipos de Procesos

  • Usuario: creados por usuario real, modo usuario.
  • Demonio: no asociados a usuario, tareas periodicas (cron, redes...).
  • Nucleo: codigo del kernel, siempre en modo nucleo.

ps: Monitorizacion de Procesos

ps aux # Todos los procesos, todos los usuarios ps al # Formato largo ps aux | grep apache

Estados de Procesos (STAT)

Sistemas de Ficheros y Discos
HDD, SSD, particiones, bloques, VFS, asignacion
▶TeoriaTEORIA

HDD vs SSD vs NVMe

  • HDD: Platos magneticos, cabezal, pistas, cilindros, sectores. Acceso mecanico.
  • SSD: Memoria NAND flash, sin partes moviles, mas rapido.
  • NVMe: SSD con interfaz PCIe directa, maxima velocidad.

Sistema de Archivos (SA/SF)

Organiza la informacion en dispositivos de almacenamiento. Division logica: Particiones → Agrupaciones → Bloques.

Tamano de Bloque

  • Pequeno: mas operaciones E/S, menor fragmentacion.
  • Grande: menos operaciones E/S, mayor fragmentacion.
  • Ver con stat -f .

Estructura del Sistema de Archivos

  • Bloque de carga (boot/VBR): codigo de arranque de la particion.
  • Superbloque: caracteristicas del SA (tamano, posiciones, etc). Copias de seguridad en varias posiciones del disco.
  • Mapas de bits: indican bloques y nodos-i libres.
  • Descriptores (nodos-i): metadatos de cada archivo (tamano, permisos, apuntadores a bloques).
  • Bloques de datos: donde se almacena la informacion.
sudo dumpe2fs -h /dev/nvme0n1p3 # Info superbloque stat /etc/passwd # Info nodo-i ls -i /etc/passwd # Ver nodo-i

Servidor de Archivos (4 capas)

  1. VFS (Virtual File System): interfaz comun para todos los SA.
  2. Modulo de organizacion: traduce direcciones logicas a fisicas.
  3. Servidor de bloques: E/S de bloques + cache.
  4. Manejadores de dispositivos: especificos por hardware.

Asignacion de Bloques

  • Contigua: bloques seguidos (simple pero fragmentacion externa).
  • Enlazada (FAT): cada bloque apunta al siguiente.
  • Indexada (nodos-i): apuntadores directos + indirectos (simple, doble, triple).

Gestion del Espacio Libre

  • Mapas de bits: un bit por bloque (0=libre, 1=ocupado).
  • Listas de recursos libres.

▶TeoriaTEORIA

Montaje y Desmontaje

Montar = anadirlo al arbol de directorios. Desmontar = dejarlo consistente y desconectarlo.

mount /dev/sdb1 /punto/montaje mount LABEL=etiqueta /punto/montaje mount UUID=a244... /punto/montaje umount /punto/montaje

/etc/fstab

Configuracion de montaje permanente. Se lee al arrancar.

Receta: Anadir un Disco

  1. Apagar, conectar disco, encender.
  2. lsblk para identificar (ej: /dev/sdb).
  3. Particionar: sudo fdisk /dev/sdb o cfdisk.
  4. Crear SF: sudo mkfs -t ext4 -L etiqueta /dev/sdb1
  5. Crear punto de montaje: sudo mkdir /punto/montaje
  6. Montar: sudo mount /dev/sdb1 /punto/montaje
  7. Para permanente: editar /etc/fstab.

Comprobacion SF

fsck /dev/sda1 # Comprobar (particion desmontada!) e2fsck -f /dev/sda1 # Forzar comprobacion ext

Journaling

ext3/ext4 usan journaling: registro de transacciones pendientes para recuperacion rapida tras caida.

Creacion de Sistemas de Ficheros

mkfs -t ext4 /dev/sdb1 mkfs -t xfs /dev/sdb1 mkfs -t btrfs /dev/sdb1

Planes de Prevencion de Catastrofes

El administrador debe: planear, implementar copias periodicamente, guardar copias en lugar seguro.

Determinar: que copiar, donde, como y cuando.

Escenarios de Perdida

  • Errores humanos (borrar ficheros por accidente).
  • Virus y software destructivo.
  • Personas malintencionadas.
  • Fallos del hardware.

Estrategias de Backup

TipoDescripcionVentajaDesventaja
CompletaTodo el sistemaRestauracion rapidaLenta, mucho espacio
DiferencialCambios desde la ultima completaRapida restauracionCrece con el tiempo
IncrementalCambios desde la ultima copia (cualquiera)Rapida, poco espacioRestauracion lenta

Herramientas

Gestion de las Comunicaciones
Red, NFS, NIS, SAMBA, servicios de red
▶TeoriaTEORIA

Conceptos Basicos de Red

  • Interfaces de red: conexiones fisicas/virtuales (eth0, wlan0).
  • IP interna (privada): 192.168.x.x, 10.x.x.x. No accesible desde Internet.
  • IP externa (publica): identificador unico asignado por el ISP.
  • DNS: traduce nombres de dominio a IPs.
  • Gateway: punto de acceso entre redes.
  • Puerto: numero que identifica servicios (80=HTTP, 443=HTTPS, 22=SSH).

Configuracion y Diagnostico

ip -a # Ver interfaces e IPs ping host # Test conectividad traceroute host # Ruta de paquetes ss -tulnp # Puertos abiertos curl ifconfig.me # IP externa

Ficheros de Configuracion

  • /etc/hostname - Nombre del equipo.
  • /etc/hosts - Resolucion local de nombres.
  • /etc/resolv.conf - Servidores DNS.
  • /etc/network/interfaces - Configuracion de red (Debian).

Servicios de Red

systemctl --type=service --state=running # Listar servicios activos

Autenticacion Unica (SSO)

  • NIS (Network Information Service): comparte informacion de usuarios/grupos en red.
  • LDAP: directorio centralizado para autenticacion.

NFS (Network File System)

Sistema de ficheros distribuido. Permite montar directorios remotos.

# SERVIDOR: /etc/exports /home 192.168.1.0/24(rw,sync,no_subtree_check) sudo exportfs -ra sudo systemctl restart nfs-kernel-server # CLIENTE: sudo mount servidor:/home /mnt/home_remoto

SAMBA

Interoperabilidad con Windows (compartir ficheros e impresoras).

Configuracion: /etc/samba/smb.conf

sudo smbpasswd -a usuario # Crear usuario Samba sudo systemctl restart smbd
